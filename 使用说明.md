# 使用说明

## 1. 启动应用

推荐在项目目录执行：

```bash
npm install
npm run dev
```

启动后打开 `http://localhost:3000`。后端 API 默认运行在 `http://localhost:3006`。

Windows 下也可以双击 `start.bat`。

## 2. 选择图片目录

在“目录与扫描”区域点击“选择”，输入需要处理的图片文件夹路径。

示例：

```text
D:\图片测试
D:\照片
D:\工作\图片素材
```

选择后点击“扫描图片”，应用会列出符合格式条件的图片。

## 3. 设置扫描格式

在“格式筛选”区域勾选需要处理的格式：

- PNG
- JPG
- JPEG
- WebP
- AVIF

未勾选的格式不会进入扫描结果。

## 4. 设置输出参数

在“输出设置”中可以调整：

- 压缩模式：无损优先、低损高画质、均衡压缩、极限压缩。
- 输出格式：保留原始格式，或转换为 PNG、JPG、JPEG、WebP、AVIF。
- 宽度和高度：留空表示保持原尺寸。
- 保持宽高比：建议默认开启。
- 跳过变大输出：建议默认开启，避免生成比原图更大的文件。
- 保留元数据：需要 EXIF 或色彩配置时开启；开启后文件可能变大。
- 直接写入源目录：开启后会写入源目录，使用前请确认已经备份。
- 输出目录：留空时写入源目录下的 `__compressed`。
- 压缩质量：数值越高画质越好，文件通常也越大。

## 5. 选择压缩模式

- **无损优先**：适合图标、透明 PNG、需要尽量保留原始画质的素材。JPG 不缩放且保持 JPG 输出时会保留原图，避免二次损失。
- **低损高画质**：适合照片、商品图、作品集素材。
- **均衡压缩**：适合大多数日常批量处理。
- **极限压缩**：适合对体积极敏感、可接受画质下降的场景。

## 6. 备份图片

如果需要保留原图，先选择备份目录，再点击“先备份”。

备份会复制源目录中的图片文件，并尽量保留原有目录结构。

## 7. 开始压缩

扫描完成后点击“开始压缩”。

压缩完成后可以在“压缩结果”中查看：

- 成功、跳过或失败状态。
- 原始大小。
- 输出大小。
- 节省比例。
- 跳过原因或失败原因。

## 8. 查看日志

“操作日志”会记录扫描、备份、压缩和错误信息。

如果日志太多，可以点击“清空”。

## 9. 常见问题

### 无法连接本地服务

确认已经执行 `npm run dev`，并且后端没有启动失败。默认后端端口是 `3006`。

### 文件夹不存在

检查输入路径是否正确，Windows 路径可以使用反斜杠，例如 `D:\照片`。

### 没有权限读写文件

请确认当前用户有源目录和输出目录的读写权限，且图片没有被其他软件占用。

### 不想覆盖原图

保持“直接写入源目录”关闭，并设置输出目录，或使用默认的 `__compressed` 输出目录。

### 为什么有些图片被跳过

默认开启“跳过变大输出”。当生成文件比原始文件更大时，工具会跳过该文件并保留原图。
