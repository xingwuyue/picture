# 图片压缩工具

一个本地运行的批量图片压缩工具，适合处理站点素材、照片目录和需要批量转格式的图片文件夹。

## 功能

- 扫描指定目录下的 PNG、JPG、JPEG、WebP、AVIF 图片。
- 批量压缩图片并显示压缩前后体积。
- 支持保留原格式，或转换为 PNG、JPG、JPEG、WebP、AVIF。
- 支持四种压缩模式：无损优先、低损高画质、均衡压缩、极限压缩。
- 支持输出变大保护，生成文件比原图更大时自动跳过。
- 可保留图片元数据，适合对色彩配置和 EXIF 有要求的场景。
- 可设置输出宽度、高度、是否保持宽高比和压缩质量。
- 可选择备份目录，在压缩前复制原始图片。
- 可指定输出目录；不指定时默认写入源目录下的 `__compressed`。

## 压缩模式

- **无损优先**：PNG/WebP/AVIF 使用无损编码；JPG 在不缩放且保持 JPG 输出时不重编码，避免二次损失。
- **低损高画质**：适合照片和生产素材，JPG 使用高质量和 4:4:4 色度采样，WebP/AVIF 使用高画质设置。
- **均衡压缩**：使用质量滑杆，兼顾体积和画质。
- **极限压缩**：追求更小体积，允许更明显的画质损失。

## 环境要求

- Node.js 16 或更高版本。
- Windows、macOS 或 Linux。
- 能访问本地端口 `3000` 和 `3006`。

## 启动

安装依赖：

```bash
npm install
```

启动前端和后端：

```bash
npm run dev
```

默认地址：

- 前端：`http://localhost:3000`
- 后端 API：`http://localhost:3006`

Windows 用户也可以双击 `start.bat`，脚本会安装依赖、启动服务并打开浏览器。

## 测试和构建

```bash
npm test
npm run build
```

构建产物会输出到 `dist/`，该目录默认不提交到 Git。

## 使用流程

1. 在“目录与扫描”中选择图片目录。
2. 在“格式筛选”中勾选需要扫描的图片格式。
3. 点击“扫描图片”，确认待处理列表。
4. 在“输出设置”中选择压缩模式、输出格式、尺寸、质量和输出目录。
5. 如需保留原图，先选择备份目录并点击“先备份”。
6. 点击“开始压缩”，在“压缩结果”和“操作日志”中查看结果。

## 常见问题

### 前端打开但无法扫描

确认后端服务已启动。默认后端地址是 `http://localhost:3006`。

### 端口被占用

修改 `vite.config.ts` 中的前端端口，或释放本机 `3000`、`3006` 端口。

### 压缩后文件没有变小

某些图片已经被充分优化。默认开启“跳过变大输出”，如果生成文件比原图更大，结果会显示为“跳过”并保留原图。

### JPG 能完全无损压缩吗

JPG 严格无损优化通常需要 `jpegtran/mozjpeg` 这类专用工具。当前无损模式会在保持 JPG 且不缩放时直接复制原图，避免二次重编码造成损失。

### GitHub 推送失败

当前远程地址配置为：

```bash
https://github.com/xingwuyue/picture.git
```

如果推送时出现认证错误，请通过 Git Credential Manager 登录 GitHub，或使用有效的 Personal Access Token。
